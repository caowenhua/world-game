# 自我迭代机制 v1.0

> 每一次皇上反馈，都是太子必须立刻解决的核心问题。

---

## 核心原则

1. **皇上说一不二** - 任何"差不多"都是罪
2. **主动反省** - 不能等问题严重了才修复
3. **记录所有问题** - 每个问题都要追踪到解决
4. **极致标准** - 这是10万日活的产品，不是玩具

---

## 迭代流程

```
皇上反馈
    ↓
太子立刻确认问题（不超过5分钟）
    ↓
分析根本原因
    ↓
制定修复方案
    ↓
执行修复
    ↓
自检（至少3轮）
    ↓
推送GitHub
    ↓
通知皇上测试
```

## 自检清单（每次修改后必须执行）

### 视觉检查
- [ ] 页面加载后能看到内容（不是空白）
- [ ] 所有UI元素正常显示
- [ ] 动画流畅不卡顿
- [ ] 手机端适配

### 功能检查
- [ ] 登录/注册正常
- [ ] 角色选择正常
- [ ] 游戏地图正常渲染
- [ ] 角色移动正常
- [ ] 战斗系统正常
- [ ] 卡牌系统正常

### 核心检查
- [ ] 卡牌是剧情驱动（不是技能卡）
- [ ] 发牌员是全场景机制
- [ ] 世界观和剧情文档一致

---

## 皇上核心要求（永远不能忘）

### 1. 卡牌推进世界
- 不是战斗累了抽卡休息
- 是"每张卡都是世界的一个碎片"
- 抽到"古龙的鳞片" → 世界告诉你古龙的传说
- 击败"亡灵巫妖" → 获得"觉醒者之心"传说卡

### 2. Pokemon像素风
- 地图要有像素质感
- 角色要有精灵动画
- 地形要有细节纹理
- UI要精美

### 3. 发牌员是机制不是人
- 战斗后 → 可能掉卡
- 探索时 → 可能发现卡
- NPC对话 → 可能获得卡
- 精英怪 → 必掉传说卡
- 宝箱 → 可能开出卡
