## 一、游戏概述

**一句话描述**：一款Pokemon像素风地图 + 卡牌驱动剧情的开放世界RPG，玩家通过收集事件卡牌推进《千年契约》的宏大叙事。

**差异化特点**：
- Pokemon式精美像素地图（Canvas 2D渲染）
- 8种卡牌类型驱动世界运转
- 发牌员全场景机制（NPC、公告板、营地、商人）
- AI-ready架构，支持实时内容生成

### 3.3 发牌算法

```typescript
function drawRandomCardFromDealer(dealer: Dealer): EventCard | null {
  const cards = getCardsFromDealer(dealer);
  if (cards.length === 0) return null;
  
  // 稀有度权重：稀有度越低（数字越小）权重越高
  const weights = cards.map(c => 6 - c.rarity);
  const totalWeight = weights.reduce((a, b) => a + b, 0);
  
  let random = Math.random() * totalWeight;
  for (let i = 0; i < cards.length; i++) {
    random -= weights[i];
    if (random <= 0) return cards[i];
  }
  return cards[cards.length - 1];
}
```

### 3.4 交互规则

1. **靠近检测**：玩家与发牌员距离 ≤ 1.5格时触发交互提示
2. **按键触发**：按 `E` 键打开发牌员界面
3. **卡牌展示**：显示该发牌员支持的所有卡牌类型
4. **抽卡**：从卡池中随机抽取一张卡（按稀有度权重）

## 五、战斗系统设计

### 5.1 回合制自动战斗

**玩家行动**：
1. **攻击（J键）**：基础伤害 = 攻击力 × 攻击帧动画
2. **闪避（空格）**：向面朝方向移动3格，本回合无法攻击
3. **互动（E键）**：与发牌员交互、打开宝箱

**敌人AI状态机**：
```
patrol → chase → attack
  ↑        ↓
  └──←←←←←┘
```

- **patrol**：沿路径巡逻
- **chase**：发现玩家（距离<6），追击
- **attack**：距离<1.2时攻击玩家

### 5.2 伤害公式

```
实际伤害 = max(1, (攻击方.攻击力 - 防御方.防御力/2) × 随机系数(0.8-1.2))
```

### 5.3 经验值与升级

| 等级 | 所需经验 | HP上限增量 | 攻击增量 | MP上限增量 |
|------|---------|-----------|---------|-----------|
| 1→2 | 100 | +20 | +5 | +10 |
| 2→3 | 150 | +20 | +5 | +10 |
| 3→4 | 225 | +20 | +5 | +10 |
| ... | ... | ... | ... | ... |

### 5.4 精英怪物

精英怪物（isElite）特征：
- 更高的属性（HP×1.5, ATK×1.5）
- 必定掉落3⭐以上卡牌
- 名称前显示"精英"标签
- 显示★标记

## 八、游戏状态机

```
story → playing → gameover/victory
         ↓
    showCardGame (按E)
         ↓
    showRewards (抽卡)
```

- **story**：开场剧情对话
- **playing**：主游戏循环
- **gameover**：玩家HP归零
- **victory**：击败最终BOSS（本版本暂未实现）
- **showCardGame**：发牌员界面
- **showRewards**：卡牌获得弹窗

## 十、卡牌获取途径

1. **击败普通怪物**：30%概率掉落1-3⭐卡牌
2. **击败精英怪物**：100%概率掉落3-5⭐卡牌

## 十一、设计原则

### 黄金法则（来自PRD）
1. **成本透明**：每张卡牌显示获取成本
2. **交付明确**：故事内容清晰可见
3. **节奏可控**：卡牌类型平衡，叙事节奏良好
4. **可追溯性**：每张卡牌都有唯一ID和类型

### 卡组构建理念
- 卡牌是剧情的碎片，推进世界用的
- 玩家不需要收集所有卡牌，但收集能带来加成
- 不同发牌员提供不同类型卡牌，鼓励探索
