# AI World Game - 产品需求文档（PRD）

> **项目代号**：羊蹄山之魂·AI版  
> **目标用户**：16-35岁游戏玩家、AI爱好者、TRPG玩家  
> **DAU目标**：10万+  
> **版本**：v1.0  
> **日期**：2026年4月3日

---

## 一、产品愿景

**一句话**：一款用AI实时生成无限内容的开放世界RPG，玩家在虚拟世界中自由探索，每一次冒险都是独一无二的叙事体验。

**差异化**：
- 传统RPG：内容是有限的，最终会消耗完
- 本游戏：AI实时生成，内容永不枯竭，每次游玩都是新故事
- 核心壁垒：事件卡组 + LLM生成 = 可控的无限内容

**核心体验**：
> 玩家感受到的不是"玩游戏"，而是"活在一个活着的世界里"

| 用户类型 | 占比 | 特征 | 吸引点 |
|---------|------|------|--------|

- 想要更个性化的故事体验

## 三、核心功能

### 3.1 事件卡组系统（核心）

**作用**：控制AI生成内容的呈现节奏，确保体验流畅

**卡组结构**：
```
每轮卡组 = 12张卡
- 主线剧情卡: 2张（推动核心叙事）
- 支线故事卡: 3张（本地故事、角色背景）
- 探索发现卡: 2张（隐藏地点、新NPC）
- 挑战遭遇卡: 2张（战斗、谜题、抉择）
- 成长奖励卡: 1张（装备、能力提升）
- 氛围营造卡: 1张（孤寂、温暖、紧张）
- 空白卡: 1张（留白、呼吸空间）
```

**卡组规则**：
1. 线性顺序，但可根据玩家状态动态调整
2. 紧急卡牌可跳到最前（如：生命值低时优先出治疗卡）
3. 卡组耗尽后重新开始新循环，但不洗牌
4. 每轮循环增加卡组丰富度（解锁更多卡牌类型）

### 3.2 LLM动态内容生成

**生成内容类型**：

| 内容类型 | 描述 | LLM输入 | LLM输出 |
|---------|------|---------|---------|
| 剧情对话 | NPC台词 | 角色+情境+卡牌类型 | 对话文本 |
| 场景描述 | 地点描写 | 区域+氛围目标 | 描写文字 |
| 物品描述 | 物品/任务物品 | 物品类型+背景 | 名称+描述 |
| 事件叙事 | 事件叙述 | 事件类型+玩家状态 | 叙述文字 |
| 敌人背景 | 敌人介绍 | 敌人类型+区域 | 背景故事 |

**质量控制**：
- 卡组提供结构，LLM提供内容
- 预设安全边界（不允许暴力、成人内容）
- 置信度低于阈值时使用预设内容兜底

## 四、游戏流程

### 4.1 创建角色

1. 输入角色名称
2. 选择或AI生成背景故事
3. 选择性格特征（影响叙事风格）
4. AI生成初始状态和起始事件

### 4.2 探索循环

```
主循环：
1. 玩家移动/选择行动
2. 系统检测位置和交互对象
3. 根据位置和状态决定发牌员
4. 发牌员触发 → 从卡组抽牌
5. 决定事件类型 → 调用LLM生成内容
6. 交付事件 → 更新状态
7. 检查状态变化 → 触发新事件/解锁内容
8. 渲染体验 → 玩家感知到"活着的世界"
```

### 4.3 核心交互类型

| 交互类型 | 触发方式 | 体验 |
|---------|---------|------|
| 对话 | 与NPC交谈 | AI生成对话 |
| 探索 | 移动到新地点 | AI生成场景描写 |
| 战斗 | 遭遇敌人 | 回合制+LLM描述 |
| 任务 | 接取/完成任务 | 动态任务链 |
| 收集 | 获得物品 | AI生成物品描述 |
| 抉择 | 关键选择 | 分支剧情 |

## 五、技术架构

### 5.1 技术栈

| 层 | 技术选型 | 理由 |
|---|---------|------|
| 前端 | Next.js 14 + React | SSR、SEO、优秀UX |
| UI框架 | TailwindCSS + shadcn/ui | 快速、美观 |
| 动画 | Framer Motion | 流畅动画体验 |
| 后端 | Go + Fiber | 高性能、并发强 |
| 数据库 | MongoDB | 灵活文档存储 |
| 缓存 | Redis | 会话、排行榜 |
| LLM | MiniMax / DeepSeek | 成本效益比高 |
| 协议 | gRPC + Protobuf | 高效、类型安全 |
| 实时 | WebSocket | 推送事件 |
| 部署 | Docker + K8s | 可扩展 |
